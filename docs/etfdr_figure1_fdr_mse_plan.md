# ETFDR Figure 1 重点阅读与复现方案（聚焦 FDR-MSE）

> 范围说明：本文档只基于 `ETFDR.pdf` 的 Figure 1 及其紧邻方法段落整理，不展开全文细节。

## 1. Figure 1 的模型设置（论文原文关键信息）

- 任务：Lasso 路径上比较预测误差（CV MSE）与变量选择错误（FDR）。
- 数据：高斯线性模型，样本量 `n = 600`，变量维度 `d = 200`。
- 信号：`d1 = 20` 个真信号，且信号强度相同且中等。
- 两个场景：
  - 场景 (a)：解释变量独立。
  - 场景 (b)：解释变量高相关，按 AR(0.8) 相关结构生成。
- 图上元素：
  - 蓝线：CV MSE 曲线。
  - 黑线：真实 FDR（仅模拟可得）。
  - 红线：论文提出的 FDR 估计量。
  - 竖线：最小 CV 与 one-standard-error 规则位置。

## 2. 本文采用的 FDR 计算/估计方法（Figure 1 相关）

### 2.1 真值 FDR（用于模拟评估）

- 先定义 FDP：
  - `FDP = |R ∩ H0| / |R|`，当 `|R| = 0` 取 0。
- 真值 FDR：
  - `FDR = E[FDP]`（对数据重复取期望）。

### 2.2 论文的 FDR 估计器（红线）

论文把总体 FDR 拆成变量贡献后估计，核心形式可写成：

- `FDR_hat = Σ_j FDR_hat_j`
- `FDR_hat_j ≈ E_{Hj}[ I{j∈R}/|R| | S_j ] × I{p_j > c}/(1-c)`

其中：
- `R`：当前选择方法（如 Lasso 在某个 λ 下）的选中变量集；
- `S_j`：在高斯线性模型下与第 j 个零假设相配的充分统计量；
- `p_j`：第 j 个变量的双侧 t 检验 p 值；
- 论文示例常用阈值 `c = 0.1`。

论文强调该估计器在其建模假设下对真 FDR 具有保守性（非负偏差）。

## 3. 方法优劣（针对 Figure 1 这个用途）

### 优点

- 能把“变量选择质量”显式叠加到 CV MSE 旁边，揭示“预测好但误选多”的区间。
- 对选择算法较通用（不仅限 Lasso，理论上可接 FS/LARS/ENet 等）。
- 在模拟中可与真 FDR 对照，便于校准和解释。

### 局限

- 依赖模型假设与检验构造（如线性高斯 + t 检验/充分统计量）；偏离假设时可能偏保守或波动变大。
- 需要额外计算，且标准误估计通常要 bootstrap，开销高于单纯 CV。
- 在强相关场景下，低 FDR 与低 MSE 可能天然冲突，方法更多是“揭示冲突”，不是“消除冲突”。

## 4. 复现 Figure 1 的最小可行方案

## 4.1 数据与路径

- 固定：`n=600, d=200, d1=20`。
- 设计矩阵：
  - 场景 (a)：`Σ = I_d`。
  - 场景 (b)：`Σ_{jk} = 0.8^{|j-k|}`。
- 真实系数：前 `d1` 个非零，其余为 0；非零强度用单一固定值（后续再做敏感性）。
- 候选复杂度：Lasso 的 `λ` 路径（建议 80~100 点，log 均匀）。

## 4.2 评估量

- `CV MSE(λ)`：5-fold。
- `True FDR(λ)`：模拟中已知真信号集，直接算 FDP 并跨重复取均值。
- `FDR_hat(λ)`：按论文估计器计算。
- 辅助点：`λ_cv_min` 与 `λ_1se`。

## 4.3 输出图

- 每场景一张图，x 轴为 `-log(λ)`。
- 左 y 轴：CV MSE；右 y 轴：FDR。
- 同图叠加 `CV MSE / True FDR / FDR_hat` 与两条竖线。

## 5. 扩展：把 CVC 与 CV 的 FDR-MSE 画在一起比对（可行性）

结论：可行，但需明确“CVC 作用在 λ 选择层面”。建议做两层输出。

### 层 A：路径层对照（推荐主图）

- 对每个 `λ` 同时计算：`CV MSE(λ)`、`True FDR(λ)`、`FDR_hat(λ)`。
- 在同一坐标上标出：
  - `λ_cv`（CV 最小或 1se）；
  - `λ_cvc`（CVC 规则选出的 λ）。
- 这样能直接看到 CVC 与 CV 在“同一路径上的位置差异”。

### 层 B：点估计层对照（补充表/图）

- 只比较 `λ_cv` 与 `λ_cvc` 两个点：
  - `MSE@λ_cv` vs `MSE@λ_cvc`
  - `FDR@λ_cv` vs `FDR@λ_cvc`
- 跨重复给均值与标准差（或误差棒）。

## 6. 可行性评估与建议设置

- 计算可行：高维度不高（d=200），R + glmnet 可完成。
- 主要额外成本：`FDR_hat` 与标准误（若加 bootstrap）。
- 建议分阶段：
  1. 先无 bootstrap，跑 `reps=50` 验证图形趋势；
  2. 再扩到 `reps=200`；
  3. 最后按需要补标准误带。
- 对接现有仓库最小增量：
  - 新增 `scripts/run_etfdr_fig1.R`
  - 新增 `configs/etfdr_fig1_config.R`
  - 结果落在 `results/etfdr_fig1_*` 与 `figures/etfdr_fig1_*`。
